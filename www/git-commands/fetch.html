git fetch

The git fetch command downloads commits, files, and refs from a remote repository into your local repository without merging them into your current branches. It lets you see what others have been working on before you integrate changes.

Common usages

  • Fetch from the default remote:

    git fetch

  • Fetch from a specific remote (e.g., origin):

    git fetch origin

  • Fetch and prune deleted remote branches:

    git fetch --prune

  • Fetch from all remotes and prune:

    git fetch --all --prune

Tips

  • After fetching, use git log ..origin/main or git diff ..origin/main to inspect changes.
  • Combine with git merge or git rebase to integrate fetched changes.
